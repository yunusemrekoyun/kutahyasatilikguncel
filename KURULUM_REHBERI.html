Genel Bakis

Teslim paketi nasil calisir?

Kaynak kod Git ile tasinir. Gizli ayarlar, veritabani ve kullanici gorselleri ayri tasinir.

01Git deposunu clone et.
02.env dosyasini olustur.
03Bagimliliklari ve veritabanini kur.
04Uygulamayi calistir ve dogrula.
Onemli: .env, SQLite veritabani ve yuklenen medya dosyalari Git deposuna dahil edilmez. Bunlar gerekiyorsa ayri ve guvenli bicimde teslim edilmelidir.
Gereksinimler

Yeni bilgisayarda gerekenler

Node.js

Node.js 22 LTS ve beraberinde gelen npm.

node -v npm -v

Git

Depoyu clone etmek ve guncellemeleri almak icin.

git --version

SQLite CLI

Uygulama icin zorunlu degildir; inceleme ve yedekleme icin faydalidir.

sqlite3 --version
Uygulama SQLite'a better-sqlite3 paketiyle baglanir. Ayrica calisan bir SQLite servisi kurulmaz.
Hizli Kurulum

Sifirdan demo kurulumu

Terminal
git clone GIT_REPO_ADRESI cd kutahyasatilik.com npm ci # Sonraki slayttaki icerikle .env dosyasini olustur. mkdir -p public/uploads npm run setup:demo npm run dev
Tarayicida http://localhost:3000 ve yonetim paneli icin http://localhost:3000/admin acilir.
npm run setup:demo ornek ilanlari yeniden olusturur. Gercek veri bulunan bir veritabaninda bu komutu kullanmayin.
Ortam Ayarlari

Yerel bilgisayar icin .env

Proje kokunde .env adinda tek dosya olusturun.

DATABASE_URL="file:./dev.db" UPLOAD_DIR="./public/uploads" AUTH_SECRET="EN_AZ_32_KARAKTER_RASTGELE_BIR_DEGER" ADMIN_EMAIL="admin@example.com" ADMIN_PASSWORD="GUCLU_BIR_SIFRE" NEXT_PUBLIC_SITE_URL="http://localhost:3000" NEXT_PUBLIC_PHONE="+90 555 555 55 55" NEXT_PUBLIC_WHATSAPP="905555555555" NEXT_PUBLIC_EMAIL="info@example.com" NEXT_PUBLIC_GA_ID="" NEXT_PUBLIC_GTAG_ID="" NEXT_PUBLIC_ADS_CONVERSION_LABEL=""
Secret uretmek icin: node -e "console.log(require('crypto').randomBytes(48).toString('base64url'))"
Veri Secenegi A

Yeni ve bos veritabanina baslamak

1. Semayi kur

npx prisma migrate deploy

Migration dosyalari yeni dev.db icinde tablolari olusturur.

2. Demo verisini ekle

npm run seed

Admin, ilce verileri ve ornek ilanlar eklenir.

Seed islemi mevcut ilan gorsellerini ve ilanlari temizleyip ornek ilanlari yeniden ekler. Yalnizca ilk demo kurulumunda kullanin.
Veri Secenegi B

Mevcut veritabani ve gorselleri tasimak

SQLite verisi

Kaynak bilgisayardaki veya VPS'teki prod.db dosyasini hedef bilgisayara kopyalayin.

cp TESLIM_EDILEN_PROD_DB ./dev.db

Medya dosyalari

Upload klasorunun tamamini hedefteki klasore kopyalayin.

rsync -av TESLIM_UPLOADS/ public/uploads/
Mevcut DB tasindiysa npm run seed calistirmayin. Yalnizca npx prisma generate ve npm run dev yeterlidir.

Git Yayinlama

Depoyu ilk kez pushlamak

git add -A git status git commit -m "Prepare production-ready real estate portal" git branch -M main git remote add origin GIT_REPO_ADRESI git push -u origin main
Remote daha once tanimliysa git remote add yerine git remote set-url origin GIT_REPO_ADRESI kullanin.
Push oncesi GitHub/GitLab dosya listesinde .env, veritabani veya yuklenen fotograflar gorunmemelidir.

VPS Kurulumu

Ilk production kurulumu

sudo mkdir -p /var/www/kutahyasatilik.com/{app,data,uploads,logs,backups} cd /var/www/kutahyasatilik.com/app npm ci npx prisma migrate deploy # Yalnizca ilk demo verisi gerekiyorsa: npm run seed npm run build cp -r public .next/standalone/ mkdir -p .next/standalone/.next cp -r .next/static .next/standalone/.next/
Uygulama servisi node .next/standalone/server.js ile calisir. Production'da systemd ile yonetilmesi onerilir.
VPS Guncelleme

Yeni kodu canliya almak

Kaynak bilgisayar
rsync -az --delete \ --exclude node_modules \ --exclude .next \ --exclude .git \ --exclude .env \ --exclude dev.db \ --exclude public/uploads/ \ ./ root@SUNUCU_IP:/var/www/kutahyasatilik.com/app/
VPS
cd /var/www/kutahyasatilik.com/app npm ci npx prisma migrate deploy npm run build cp -r public .next/standalone/ mkdir -p .next/standalone/.next cp -r .next/static .next/standalone/.next/ sudo systemctl restart kutahyasatilik
Guncelleme sirasinda npm run seed calistirmayin. Gercek ilanlari ornek veriyle degistirebilir.

Sorun Giderme

Yaygin hatalar

Prisma Client bulunamadi

npx prisma generate

Tablo bulunamadi

npx prisma migrate deploy npx prisma migrate status

Gorsel 404

echo "$UPLOAD_DIR" ls -lah "$UPLOAD_DIR" sudo systemctl restart kutahyasatilik

Port dolu

sudo ss -tulpn | grep 3003 sudo systemctl status kutahyasatilik
Yedekleme

Korunmasi gereken iki veri grubu

1. SQLite veritabani

sqlite3 /var/www/kutahyasatilik.com/data/prod.db \ ".backup '/var/www/kutahyasatilik.com/backups/prod-$(date +%F).db'"

2. Upload klasoru

tar -czf \ /var/www/kutahyasatilik.com/backups/uploads-$(date +%F).tar.gz \ /var/www/kutahyasatilik.com/uploads
Kod tekrar Git'ten alinabilir. Gercek kayip riski veritabani ve yuklenen gorsellerdedir; bu iki alan duzenli yedeklenmelidir.
